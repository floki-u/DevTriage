# DevTriage 桌面外壳与项目关联设计

## 目标

为已有的本地 Rust 分析引擎提供 macOS 首版桌面入口。用户可从菜单栏弹窗粘贴日志，查看简短分析结果，确认推测的项目，并主动复制已脱敏的 AI 上下文。

本阶段交付可运行的 Tauri 2 + React 桌面应用；它直接复用 `devtriage-core`，不会重新实现分析模型或通过 CLI 子进程交换数据。

## 范围

### 包含

- 常驻菜单栏入口与弹窗；
- 手动粘贴日志并调用现有 `Pipeline` 分析；
- 摘要、分析深度、项目候选和“复制 AI 上下文”；
- 单一详细窗口，展示当前问题现场、证据、脱敏转换和三档输出预算；
- 从日志路径推测项目候选，并允许手动选择文件夹；
- 本地保存最近确认项目的路径与使用时间，供后续候选排序；
- 用户可恢复的错误提示和核心逻辑测试。

### 不包含

- Git、进程、端口、运行时版本或任意项目文件读取；
- 自动复制、原始日志保存、分析历史、完整归档和联网；
- Windows、Linux 或菜单栏以外的常驻交互；
- 社区规则、额外语言深度能力包和加密存储。

## 交互

菜单栏点击后打开快速弹窗。用户粘贴日志后，应用立即运行分析并展示：首要错误摘要、分析深度、候选项目及其匹配原因，以及“复制 AI 上下文”按钮。复制始终由用户触发。

候选项目按以下信号排序：日志中的绝对路径前缀匹配优先，其次为相对路径与最近项目匹配，最后为最近确认时间。用户确认候选或手动选择文件夹后，应用只保存该路径和确认时间。

“打开详细视图”创建或聚焦单一窗口。该窗口读取同一份内存中的当前问题现场，允许在紧凑、标准、详细预算之间切换，并展示证据、来源、转换和输出预览。

## 架构

新增独立 `crates/devtriage-desktop` crate：Tauri Rust 层依赖 `devtriage-core`，React 作为其前端资源。

```text
菜单栏弹窗 / 详细窗口
        │ Tauri invoke
        ▼
Desktop commands ──► Pipeline ──► IssueContext
        │                         │
        ├──► ProjectResolver ◄────┘
        └──► RecentProjects（仅路径、确认时间）
```

Rust command 负责：分析输入、解析项目候选、确认项目、读取最近项目、复制当前输出与打开详细窗口。React 仅保存界面状态和由 command 返回的当前问题现场。原始日志不写入持久化存储；重启后不恢复分析结果。

`ProjectResolver` 只能根据日志文本、用户选择的目录和最近项目记录产生候选。它不读取候选目录内容，也不访问 Git、进程或网络。

## 数据与隐私

最近项目记录的每项仅包含规范化的本地路径及最后确认时间。持久化文件保存在应用数据目录；分析现场、原始输入和复制内容只存在内存或系统剪贴板中。

所有复制输出由既有核心管线生成，因此沿用凭据、JWT 和邮箱脱敏。界面在复制前不添加未经脱敏的字段。

## 错误处理

- 剪贴板不可用时保留手动输入，并显示可关闭的提示；
- 分析失败时显示错误且保留用户输入，允许重试；
- 项目候选路径不存在、无权限或用户取消选择时不写入最近项目；
- 复制失败时显示提示，当前分析结果不丢失；
- 详细窗口创建失败时保留快速弹窗，不中断分析或复制。

## 验收与验证

- 粘贴 TypeError 样例后，快速弹窗显示摘要和主动复制按钮；
- 复制结果包含事实与请求分区，且不泄露样例秘密；
- 输入中含路径时返回解释过的项目候选；用户确认后重启应用仍可用作排序信号；
- 手动选目录可覆盖候选，但取消不产生持久化写入；
- 详细窗口展示同一现场并正确切换三档输出预算；
- Rust command、候选排序、最近项目存储及 React 高价值流程均有自动测试；
- `cargo fmt --check`、workspace 测试与严格 Clippy 通过。
